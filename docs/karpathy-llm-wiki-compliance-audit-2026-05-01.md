# Karpathy LLM-Wiki Compliance Audit — 2026-05-01

Goal: verify that Alpha-Wiki still preserves the Karpathy LLM-Wiki core while adding production discipline. This audit covers the Phase 0 baseline before deeper Phase 1a hardening.

## Standard Used

This is not an external certification standard. The compliance baseline is the Karpathy LLM-Wiki sketch as captured in this repository:

- `references/examples/karpathy-original.md`
- `references/concept.md`
- `docs/_references.md` §1
- `docs/ADR-003-no-embeddings-mvp.md`

Core requirements:

1. Simple file memory: raw sources, markdown wiki, runtime schema/instructions.
2. Three operations: `ingest`, `query`, `lint`.
3. Two service files: `index.md`, `log.md`.
4. No embeddings/vector store in MVP.
5. Agent can quickly orient through index/brief before reading detailed pages.

## Detailed Checks

### 1. Simple Markdown Memory

Karpathy baseline: wiki memory should be inspectable as plain files, not hidden in a retrieval system.

Implementation evidence:

- Wiki pages are Markdown under `<wiki_dir>/`.
- Page identity comes from frontmatter (`title`, `slug`, `status`, etc.).
- Links are `[[wikilinks]]`.
- Graph artifacts are generated files under `<wiki_dir>/graph/`.
- Bootstrap writes `README.md`, `CLAUDE.md`, `index.md`, `log.md`, and graph seeds.

Status: pass.

### 2. Three Layers

| Layer | Karpathy intent | Alpha-Wiki implementation |
|---|---|---|
| Raw | Immutable source evidence | `raw/docs`, `raw/transcripts`, `raw/chats`, `raw/web`; agent treats `raw/**` as read-only. |
| Wiki | LLM-maintained markdown | `<wiki_dir>/**` excluding `graph/`; writable through skills. |
| Schema | Agent runtime contract | `CLAUDE.md`, `.alpha-wiki/config.yaml`, gated `/alpha-wiki:evolve`. |

Status: pass.

### 3. Three Operations

| Operation | Phase 0 evidence | Notes |
|---|---|---|
| `ingest` | `skills/ingest/SKILL.md`, `commands/ingest.md` | Skill-mediated operation exists and is operationally specified. A deeper deterministic ingest pipeline remains Phase 1a. |
| `query` | `skills/query/SKILL.md`, `commands/query.md` | Reads `context_brief.md`, `index.md`, then relevant pages; no embedding search. |
| `lint` | `tools/lint.py`, `skills/lint/SKILL.md`, tests | Deterministic backend already exists. |

Status: pass for Karpathy baseline. Phase 1a should harden `ingest` and `query` with pressure fixtures and deterministic helpers where useful.

### 4. Service Files

| File | Purpose | Evidence |
|---|---|---|
| `index.md` | Agent-facing catalog / table of contents | `assets/index-md.j2`, generated by bootstrap, read by `query`, used by orphan lint. |
| `log.md` | Append-only operation diary | `assets/log-md.j2`, append instructions in skills/hooks, read by `status` and `rollup`. |

Status: pass.

### 5. Fast Context Grasp

Karpathy asks the agent to orient through the index before deep reading. Alpha-Wiki extends this with an auto-generated brief.

Evidence:

- `tools/wiki_engine.py` builds `graph/context_brief.md` with recent log, active pages, and open questions.
- `assets/hooks/session-start.sh` loads the brief at session start.
- `assets/claude-md.j2` requires session-start to load `context_brief.md`.
- `skills/query/SKILL.md` starts retrieval with `context_brief.md`, then `index.md`.
- `tools/status.py` refreshes graph artifacts before reporting.

Status: pass.

Phase 1a improvement: add context budget profiles (`small`, `medium`, `large`) only after the core brief remains stable.

### 6. No Embeddings MVP

Evidence:

- ADR-003 locks no embeddings/no vector store for MVP.
- `pyproject.toml` has no vector DB or embedding dependency.
- Query discipline says to use filenames, slugs, wikilinks, and frontmatter instead of embedding-style guesses.

Status: pass.

## Gaps For Phase 1a

These do not block Karpathy-core compliance, but they matter for production quality:

| Gap | Why not Phase 0 blocker | Phase 1a action |
|---|---|---|
| Deterministic ingest pipeline MVP implemented | `tools/ingest_pipeline.py` writes local source pages with provenance, optional cluster links, log, graph rebuild, and lint summary. | Add pressure tests for ADR/OpenAPI/transcript/oversized/conflicting sources and resumability. |
| Deterministic query search helper implemented | `tools/wiki_search.py` provides markdown ranking and line citations without embeddings. | Add contradiction-aware pressure tests after claims tooling. |
| `index.md` population is still basic | Bootstrap creates the catalog; richer index generation can follow schema hardening. | Add deterministic index rebuild/update path. |
| Context brief is structural, not semantic | It is enough for orientation; richer summaries need stable ingest/query semantics. | Add budgeted context profiles and better page summaries. |
| Pressure tests for skills are not complete | Existing unit/integration tests cover tools and docs, not all skill pressure scenarios. | Add `tests/skills/*_pressure.py`. |
